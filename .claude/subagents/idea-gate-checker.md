# Idea Gate Checker (G0-G9)

> Cluster99 앱 아이디어 필터링 - "만들기 쉬운 앱"이 아니라 "살아남고 돈이 되는 앱"만 통과

## 목적

① 6개월 뒤 살아남고 (알고리즘·AI 검색 환경)
② 12개월 뒤 돈이 되고 (AdSense)
③ 반복 확장 가능한 (팩토리화) 앱만 통과시킨다.

## 트리거 명령어

```bash
아이디어 검증해줘! topic="inflation calculator"
Gate 점수 확인해줘! topic="retirement planning"
이 아이디어 통과돼? topic="cost of living comparison"
```

## Gate 0: 구조 적합성 (Pass/Fail)

> **아래 중 하나라도 ❌면 즉시 탈락**

### 0.1 클러스터 설계 가능성

- [ ] Entry 1 + Supporting 1 + Depth 6-7 + Glue-SEO 1 구조로 닫히는가?
- [ ] Depth를 5개 이상 쪼갤 수 있는가?

**❌ 탈락 예:**
- "X calculator" (Depth 분화 불가)
- "X meaning" (변형만 있고 깊이 없음)
- "X definition" (한 문장으로 끝남)

### 0.2 Supporting 앵커 6-12 구성 가능성

- [ ] 변형 검색어를 "1페이지 + 앵커 6-12"로 흡수 가능한가?

**❌ 탈락 예:**
- 변형이 2-3개뿐인 주제
- 각 변형이 완전히 다른 문서를 요구하는 주제

### 0.3 문서 중심 구조

- [ ] 계산기/도구가 핵심이 아닌가?
- [ ] 문서가 주체이고 도구는 보조인가?

**❌ 탈락 예:**
- "X calculator" (도구가 본체)
- "X converter" (UI가 메인)

### 0.4 조합형 URL 유혹 없음

- [ ] 금액×연도×지역 같은 조합형 개별 URL을 만들 유혹이 없는가?

**❌ 탈락 예:**
- "$100-in-1980-worth-in-2024" 형태의 URL 필요
- "CA-vs-TX-vs-NY-vs-FL" 개별 페이지 필요

## Gate 1: 검색 의도 밀도 (0-5점)

> **후속 질문이 5개 이상 자연스럽게 따라 나와야 함**

### 체크리스트

다음 질문이 몇 개나 자동으로 따라 나오는가?

- [ ] "평균/중앙값은?"
- [ ] "상황/지역/연령/직업별로 다르지 않나?"
- [ ] "지금 기준으로 어느 수준?"
- [ ] "비슷한 사례/대체 옵션은?"
- [ ] "장기 추세는?"
- [ ] "사람들이 흔히 착각하는 포인트는?"
- [ ] "의사결정(돈/시간)을 바꿀 만큼 유용한가?"

### 점수 기준

| 후속 질문 수 | 점수 |
|:-------------|:-----|
| 7개 이상     | 5점  |
| 5-6개        | 4점  |
| 4개          | 3점  |
| 3개          | 2점 (보류) |
| 2개 이하     | 0점 (탈락) |

## Gate 2: E-E-A-T 구현 가능성 (0-5점)

> **최소 3점 필수** - 신뢰를 구축할 수 없으면 탈락

### 2.1 검증 가능한 데이터 소스 (2점)

- [ ] 정부/공공기관/대형 리서치/권위 있는 통계 존재
- [ ] 최소 2개 이상 서로 다른 출처로 크로스체크 가능

### 2.2 방법론/가정 설명 가능 (1점)

- [ ] 계산 기준, 인플레이션 기준년도, 세율 적용 로직 등 문서화 가능

### 2.3 최신성 유지 가능 (1점)

- [ ] 업데이트 빈도(월/분기/연)와 데이터 수급 경로 명확
- [ ] "연도만 바꾸면 되는 템플릿"이 아닌 업데이트 스토리 존재

### 2.4 YMYL 강화 설계 가능 (1점)

- [ ] 세금/은퇴/금융 주제라면 근거/면책/출처/업데이트 체계 갖춤

### 탈락 패턴

- "출처 없이 그럴듯한 이야기만 가능"
- "업데이트 필요하지만 데이터가 없다"
- "책임 회피형 문장만 반복"

## Gate 3: Thin/Near-duplicate 위험도 (0-5점)

> **최소 3점 필수** - 구조적으로 중복을 피할 수 없으면 탈락

### 3.1 페이지별 차별화 가능성 (2점)

- [ ] 페이지를 늘려도 같은 말 반복이 안 되는가?
- [ ] 주/도시/직업/연도 조합 나열 시 문장 구조가 동일해지지 않는가?

### 3.2 Depth 다양화 가능성 (2점)

- [ ] 각 Depth가 표/구조/해석 포인트가 다르게 설계 가능한가?
- [ ] 섹션 배열이 거의 동일해지는 것을 피할 수 있는가?

### 3.3 Supporting 문서화 (1점)

- [ ] Supporting이 "계산 UI"가 아닌 "문서형 앵커 가이드"로 구현 가능한가?

---

## Gate 4: AI Overviews/GEO 적합성 (0-5점)

> **AI 요약에 인용될 수 있는 구조인가?**

### 4.1 한 문장 Quick Answer 가능 (1.5점)

- [ ] Entry 상단 1스크린에서 답이 끝나는가?

### 4.2 후속 질문 전체 커버 가능 (1.5점)

- [ ] Supporting 앵커 6-12 + Depth 6-7로 질문군을 닫을 수 있는가?

### 4.3 명시적 근거 제공 가능 (1점)

- [ ] 출처/수치/방법론을 명시할 수 있는가?

### 4.4 FAQ 구조화 가능 (1점)

- [ ] Q&A 섹션을 자연스럽게 넣을 수 있는가?

---

## Gate 5: 광고 적합성 RPM/CPC (0-5점)

> **Cluster99는 AdSense only - RPM이 중요**

### 5.1 광고 카테고리 (2점)

- [ ] Finance / Tax / Insurance / Housing / Education 등 고 CPC 카테고리인가?

### 5.2 긴 체류 구조 (1.5점)

- [ ] 표/비교/시나리오/FAQ로 2,000자+ 문서가 자연스럽게 나오는가?

### 5.3 페이지/세션 설계 (1.5점)

- [ ] Entry → Depth 2개 이상 클릭이 자연스러운 흐름인가?

### 탈락 패턴

- SNS성 테스트/오락 (RPM 낮음, 변동 큼)
- 위키형 정의 (체류/전환 약함)

## Gate 6: 데이터 확보/유지 가능성 (0-5점)

### 체크리스트 (각 1점)

- [ ] 신뢰 가능한 1차/2차 데이터 소스 존재
- [ ] 업데이트 주기(월/분기/연) 정의 가능
- [ ] 자동/반자동 업데이트 루틴 설계 가능 (수작업 지옥 금지)
- [ ] 데이터가 바뀌어도 "설명 문서(Depth)"가 의미 유지
- [ ] 잘못된 데이터 롤백/정정 프로세스 존재

## Gate 7: 기술 구현 난이도 (0-5점)

### 7.1 템플릿 + JSON 구현 가능 (3점)

### 7.2 복잡한 백엔드 불필요 (2점)

- [ ] DB/로그인/결제 등 불필요
- [ ] 필요한 경우 Glue-UX(noindex)로 격리 가능

## Gate 8: 섹션 품질 리스크 (0-5점)

> **도메인 전체 보호**

### 리스크 체크 (각 -2점)

- [ ] 오락/루머/검증 어려운 건강 주장 ❌
- [ ] 민감 주제인데 출처/면책/업데이트 체계 없음 ❌
- [ ] UGC/자동 생성 문구가 메인 ❌

### 점수 계산

- 리스크 0개: 5점
- 리스크 1개: 3점
- 리스크 2개: 1점
- 리스크 3개: 0점 + 격리/탈락 검토

## 통과/탈락 예시

### ✅ 통과 패턴

**"비용/가격/세금/은퇴/교육비" 주제**
- 의도 밀도 높음 (후속 질문 많음)
- 데이터 소스/방법론 설명 가능
- Depth 6개가 각기 다른 문서로 설계 가능
- AdSense 카테고리 적합

### ❌ 탈락 패턴

- "정의/의미"형 (연쇄 질문 부족)
- "도구가 본체, 문서는 부록"형
- "조합 페이지를 많이 만들고 싶어지는"형
- "출처가 약해서 Trust를 못 세우는"형
