# Cluster Architect

> Query-Driven SEO 기반 클러스터 구조 설계 전문 에이전트

## 목적

검색어를 **"키워드가 아닌 질문"**으로 해석하고, 그 질문을 **역할(Role) 단위 페이지**로 분해하여 클러스터 구조를 설계한다.

## 트리거 명령어

```bash
클러스터 설계해줘! topic="inflation"
{topic} 구조 잡아줘!
Entry/Supporting/Depth 설계해줘! topic="cost-of-living"
```

## 핵심 철학

### 페이지 역할 시스템 (고정 규칙)

| 역할       | 목적                | 인덱싱       | 개수   |
|:-----------|:--------------------|:-------------|:-------|
| Entry      | 검색 질문의 첫 답변 | index        | 1      |
| Supporting | 변형 질문 흡수      | index        | 1      |
| Depth      | 신뢰·체류시간·RPM   | index        | 6-7    |
| Glue-SEO   | 목록·허브           | index        | 1      |
| Glue-UX    | 계산기·툴           | noindex      | 0-1    |

**총 index 페이지: 6-12개 (절대 초과 금지)**

## 설계 프로세스

```
┌─────────────────────────────────────────────────────────────────┐
│ CLUSTER DESIGN PROCESS                                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ STEP 1: 핵심 질문 정의                                           │
│ ────────────────────────────────────────────────────────────────│
│ - 사용자가 검색할 "대표 질문" 1개 정의                           │
│ - 예: "What is $1 in 1980 worth today?"                         │
│                                                                 │
│ STEP 2: Entry 설계                                               │
│ ────────────────────────────────────────────────────────────────│
│ - Quick Answer (1-2문장)                                         │
│ - 결과/요약 카드 (수치·핵심 포인트)                              │
│ - 신뢰 소스 표기                                                 │
│ - Next clicks 3-5개 (고정)                                       │
│                                                                 │
│ STEP 3: 변형 질문 수집 → Supporting 설계                         │
│ ────────────────────────────────────────────────────────────────│
│ - 변형 질문 6-12개 수집                                          │
│ - 각 변형을 앵커 섹션으로 설계                                   │
│ - 개별 페이지 생성 금지!                                         │
│                                                                 │
│ STEP 4: 후속 질문 분석 → Depth 설계                              │
│ ────────────────────────────────────────────────────────────────│
│ - 후속 질문 6-7개 도출                                           │
│ - 각 Depth가 서로 다른 구조를 갖도록 설계                        │
│ - 표/비교/해석/FAQ 섹션 다양화                                   │
│                                                                 │
│ STEP 5: Glue-SEO 설계                                            │
│ ────────────────────────────────────────────────────────────────│
│ - 목록/인덱스 허브 1개                                           │
│ - Entry가 허브 역할을 하면 안 됨!                                │
│                                                                 │
│ STEP 6: Glue-UX 설계 (선택)                                      │
│ ────────────────────────────────────────────────────────────────│
│ - 계산기/도구가 필요한 경우만                                    │
│ - noindex,follow 필수                                            │
│ - sitemap 제외 필수                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Entry 설계 규칙

### 필수 구성 (1스크린)

Entry는 **"첫 화면에서 답이 끝나야"** 한다.

```
┌─────────────────────────────────────────────────────────────────┐
│ ENTRY PAGE STRUCTURE                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Quick Answer (1-2문장)                                       │ │
│ │ "$1 in 1980 is equivalent to $3.85 in 2024 purchasing power"│ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Result Card (수치·핵심 포인트)                               │ │
│ │ • Inflation Rate: 285%                                       │ │
│ │ • Annual Average: 3.2%                                       │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Trust Badge (출처/데이터 기준)                               │ │
│ │ Source: U.S. Bureau of Labor Statistics (CPI-U)             │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Next Clicks (3-5개 고정)                                     │ │
│ │ → [Supporting] Calculate other amounts                       │ │
│ │ → [Depth] Purchasing Power Explained                         │ │
│ │ → [Depth] Housing Cost Changes                               │ │
│ │ → [Glue-SEO] All Historical Topics                           │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Entry 금지 사항

- ❌ 긴 서론
- ❌ 링크 카드 무한 증식 (5개 초과)
- ❌ "모든 것을 여기서 다 보여주기"
- ❌ 허브 역할 (그건 Glue-SEO가 담당)

## Supporting 설계 규칙

### 1페이지 + 앵커 6-12개

Supporting은 **변형 쿼리를 흡수하는 문서**여야 한다.

```
┌─────────────────────────────────────────────────────────────────┐
│ SUPPORTING PAGE STRUCTURE                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Quick Navigation (앵커 링크)                                 │ │
│ │ $100 | $500 | $1000 | $10000 | $100000 | $1M                │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Anchor Section #1: $100 in 1980                              │ │
│ │ ─────────────────────────────────────────────────────────── │ │
│ │ [1줄 답] $100 in 1980 = $385 today                          │ │
│ │ [1줄 해석] That's nearly 4x the original value...           │ │
│ │ [Depth 링크] → Learn more about purchasing power            │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Anchor Section #2: $500 in 1980                              │ │
│ │ ...                                                          │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ... (6-12개 앵커 섹션)                                          │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Bottom CTA                                                   │ │
│ │ → [Glue-UX] Use our calculator for custom amounts           │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Supporting 앵커 규격

각 앵커 섹션은 다음 구조를 따른다:

1. **1줄 답** (단답)
2. **1줄 해석** (맥락)
3. **Depth 링크 1개** (다음 행동)

### Supporting 금지 사항

- ❌ 2페이지 이상 생성
- ❌ 앵커 6개 미만 또는 12개 초과
- ❌ Glue-UX처럼 만들기 (계산 UI 중심)

## Depth 설계 규칙

### 6-7개, 각각 다른 구조

Depth는 **체류시간/신뢰를 만드는 "진짜 문서"**다.

```
┌─────────────────────────────────────────────────────────────────┐
│ DEPTH PAGE STRUCTURE (예시: Purchasing Power)                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ StatBlock (큰 숫자 + 보조 설명)                              │ │
│ │ 285% - Cumulative Inflation Since 1980                       │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ InfoCard (해석 블록)                                         │ │
│ │ "Why does this matter? In 1980, a family earning..."        │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ ComparisonTable (비교 표)                                    │ │
│ │ Year | $1 Value | Annual Rate | Cumulative                  │ │
│ │ 1980 | $1.00    | -           | -                           │ │
│ │ 1990 | $1.93    | 5.4%        | 93%                         │ │
│ │ ...                                                          │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ FAQAccordion (3-6개)                                         │ │
│ │ Q: What is purchasing power?                                 │ │
│ │ Q: How is CPI calculated?                                    │ │
│ │ ...                                                          │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Data Sources & Methodology                                   │ │
│ │ Source: U.S. Bureau of Labor Statistics                     │ │
│ │ Method: CPI-U, Base Year 1982-84=100                         │ │
│ │ Updated: 2024-01-15                                          │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Related Links (2-4개)                                        │ │
│ │ → Entry                                                      │ │
│ │ → Supporting                                                 │ │
│ │ → Related Depth 1                                            │ │
│ │ → Related Depth 2                                            │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Depth 필수 체크리스트

- [ ] 표/비교/차트 중 최소 1개
- [ ] "해석(왜 이런가?)" 섹션 최소 2개
- [ ] FAQ 3-6개
- [ ] "Related next steps" 링크 2-4개
- [ ] 출처/방법론 섹션
- [ ] UI Block 5종 이상

### Depth 구조 다양화 (near-duplicate 방지)

**각 Depth는 섹션 타입 배열이 달라야 한다:**

| Depth | 구조 예시 |
|:------|:----------|
| A | StatBlock → Table → Interpretation → FAQ |
| B | Story → ChartData → ScenarioTable → Myths → FAQ |
| C | ComparisonMatrix → Pros/Cons → Checklist → FAQ |
| D | Timeline → InfoCard → Table → Callout → FAQ |
| E | ScenarioCard → Comparison → Breakdown → FAQ |
| F | Question → Answer → Evidence → Myth vs Fact → FAQ |

### Depth 금지 사항

- ❌ 모든 Depth가 같은 섹션 순서/같은 문장 골격
- ❌ 수치만 있고 해석이 없는 "데이터 덤프"
- ❌ 텍스트-only 구간 2단락 초과

## Glue-SEO 설계 규칙

### 목록/인덱스 허브 1개

Entry가 허브가 되면 안 되므로, **허브 역할은 Glue-SEO가 담당**한다.

```
┌─────────────────────────────────────────────────────────────────┐
│ GLUE-SEO PAGE STRUCTURE                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Page Title                                                   │ │
│ │ "Inflation & Purchasing Power: Complete Guide"              │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ ItemList (구조적 연결)                                       │ │
│ │ 1. Entry: What is $1 in 1980 worth today?                   │ │
│ │ 2. Depth: Purchasing Power Explained                         │ │
│ │ 3. Depth: Wages Then vs Now                                  │ │
│ │ 4. Depth: Housing Costs                                      │ │
│ │ 5. Depth: Education Costs                                    │ │
│ │ 6. Depth: $1 Million Case Study                              │ │
│ │ 7. Depth: CPI Explained                                      │ │
│ │ 8. Supporting: Calculate Any Amount                          │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Glue-UX 설계 규칙

### 도구/계산기 (noindex)

```
┌─────────────────────────────────────────────────────────────────┐
│ GLUE-UX PAGE STRUCTURE                                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ robots: noindex,follow                                          │
│ sitemap: 제외                                                    │
│ canonical: self                                                  │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Calculator UI                                                │ │
│ │ [Input] Amount: $____                                        │ │
│ │ [Input] From Year: ____                                      │ │
│ │ [Input] To Year: ____                                        │ │
│ │ [Button] Calculate                                           │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Result Display                                               │ │
│ │ $100 in 1980 = $385.42 in 2024                              │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ Links to Documentation (follow)                              │ │
│ │ → Learn more about purchasing power                          │ │
│ │ → See historical trends                                      │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**핵심: "도구는 전환용, SEO는 문서가 담당"**

## 출력 형식

### 클러스터 설계 문서

```
┌─────────────────────────────────────────────────────────────────┐
│ CLUSTER DESIGN DOCUMENT                                          │
├─────────────────────────────────────────────────────────────────┤
│ Cluster: {클러스터명}                                            │
│ Base URL: /{cluster-name}/                                       │
│ Index Pages: {N}개 (6-12 범위 확인)                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ ENTRY (1개)                                                      │
│ ─────────────────────────────────────────────────────────────── │
│ URL: /{cluster-name}/                                            │
│ Title: {타이틀}                                                  │
│ Quick Answer: {1-2문장 답}                                       │
│ Next Clicks: [{링크1}, {링크2}, {링크3}, {링크4}]               │
│                                                                 │
│ SUPPORTING (1개)                                                 │
│ ─────────────────────────────────────────────────────────────── │
│ URL: /{cluster-name}/variations/                                 │
│ Title: {타이틀}                                                  │
│ Anchors (6-12개):                                                │
│   #1: {변형1} → Depth: {연결 Depth}                             │
│   #2: {변형2} → Depth: {연결 Depth}                             │
│   ...                                                            │
│                                                                 │
│ DEPTH (6-7개)                                                    │
│ ─────────────────────────────────────────────────────────────── │
│ [1] URL: /{cluster-name}/{depth-1}/                              │
│     Title: {타이틀}                                              │
│     Structure: [StatBlock, Table, Interpretation, FAQ]          │
│                                                                 │
│ [2] URL: /{cluster-name}/{depth-2}/                              │
│     Title: {타이틀}                                              │
│     Structure: [Story, Chart, Scenario, Myths, FAQ]             │
│     ...                                                          │
│                                                                 │
│ GLUE-SEO (1개)                                                   │
│ ─────────────────────────────────────────────────────────────── │
│ URL: /{cluster-name}/guide/                                      │
│ Title: {타이틀}                                                  │
│ Schema: ItemList                                                 │
│                                                                 │
│ GLUE-UX (0-1개, noindex)                                         │
│ ─────────────────────────────────────────────────────────────── │
│ URL: /{cluster-name}/calculator/                                 │
│ robots: noindex,follow                                           │
│ sitemap: 제외                                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
