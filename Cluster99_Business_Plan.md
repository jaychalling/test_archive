# Cluster99 사업계획서 (2026 Updated Master Plan)

Query-Driven SEO 기반 미국 타깃 데이터 콘텐츠 플랫폼

## 1. Executive Summary (요약)

### 1.1 한 줄 정의 (업데이트)

Cluster99는 ‘검색 질문 단위(Search Query Unit)’로 설계된 고품질 콘텐츠 클러스터를 반복 확장하는 미국 시장 특화 데이터 콘텐츠 회사다.

### 1.2 우리가 해결하는 문제

2025년 이후 미국 Google 검색 환경은 다음과 같이 급격히 변했다.

- 대량 페이지 생성 SEO 모델 붕괴
- Helpful Content System의 완전 통합
- E-E-A-T(경험·전문성·신뢰) 기반 품질 신호 강화
- Thin / Near-Duplicate 콘텐츠의 구조적 패널티
- AI 요약(Search Generative Experience, AI Overviews) 등장 → 출처 신뢰도 중요성 급상승

**기존의:**

- 템플릿 페이지 대량 생성
- 계산기·도구 중심 SEO
- 키워드 조합형 페이지 전략
  은 더 이상 미국 시장에서 지속 가능하지 않다.

### 1.3 Cluster99의 해법

Cluster99는 **“페이지 수를 줄이고, 질문 단위로 역할을 분리”**하는 Query-Driven SEO 구조를 통해:

- 검색 엔진이 요구하는 품질 밀도
- 사용자가 요구하는 즉각적 답변 + 심층 해석
- 광고가 요구하는 체류시간과 신뢰도
  를 동시에 만족시키는 구조 자체를 사업 자산화한다.

## 2. 시장 환경 분석 (2025–2026 미국 SEO)

### 2.1 Google 알고리즘의 구조적 변화

**(1) Helpful Content System의 본질**

- 더 이상 “업데이트”가 아님
- 코어 랭킹 시스템에 상시 내장
- **판단 기준:**
  - 이 페이지는 사람의 질문에 실제로 답하는가
  - 다른 페이지와 대체 가능한가
  - 작성 주체의 경험·전문성·신뢰 신호가 있는가

**(2) E-E-A-T의 실질적 작동 방식**

- Trust(신뢰)가 최상위
- 단순 정보 요약 < 해석 + 맥락 + 판단 근거
- “AI가 쓸 수 있는 글” < “AI가 인용할 수 있는 글”

👉 **Cluster99의 문서형 Depth 콘텐츠는 이 변화에 최적화**

## 3. 기존 SEO 비즈니스 모델의 구조적 한계

| 기존 방식               | 한계                                   |
| :---------------------- | :------------------------------------- |
| 템플릿 기반 대량 페이지 | Near-duplicate → 사이트 전체 품질 하락 |
| 계산기/툴 중심          | UX는 좋으나 SEO 평가는 낮음            |
| 키워드 조합 페이지      | Helpful Content에 의해 노출 제한       |
| 허브형 Entry 페이지     | 검색 엔진이 의도를 해석하지 못함       |

**결론:** “많이 만드는 전략”은 이제 리스크다.

## 4. Cluster99의 핵심 전략 — Query-Driven SEO v2.0

### 4.1 Query-Driven SEO란 무엇인가

검색어를 **키워드가 아닌 ‘질문’**으로 해석하고,
그 질문을 역할(Role) 단위 페이지로 분해한다.

### 4.2 페이지 역할 시스템 (고정 규칙)

| 역할       | 목적                | 인덱싱  |
| :--------- | :------------------ | :------ |
| Entry      | 검색 질문의 첫 답변 | index   |
| Supporting | 변형 질문 흡수      | index   |
| Depth      | 신뢰·체류시간·RPM   | index   |
| Glue-SEO   | 목록·허브           | index   |
| Glue-UX    | 계산기·툴           | noindex |

👉 **역할 혼합 금지**
👉 **페이지 수가 아니라 ‘역할 정확도’가 SEO 성과를 결정**

## 5. 서비스 아키텍처 (Product Architecture)

### 5.1 도메인 전략

- 단일 메인 도메인: `cluster99.com`
- 모든 서비스는 서브디렉토리 앱으로 운영

```
cluster99.com/
 ├─ inflation/
 ├─ cost-of-living/
 ├─ retirement/
 ├─ tax-bracket/
 └─ fd-calculator/
```

### 5.2 왜 서브디렉토리인가?

- Google은 섹션 단위 품질 신호를 인식
- 앱별로:
  - 품질 관리
  - E-E-A-T 맥락 분리
  - 실패 실험 격리 가능

## 6. 클러스터 설계 원칙 (핵심 IP)

### 6.1 클러스터당 인덱스 절대량 고정

- 6–12개 인덱스 페이지 초과 금지

| 구성       | 개수    |
| :--------- | :------ |
| Entry      | 1       |
| Supporting | 1       |
| Depth      | 6–7     |
| Glue-SEO   | 1       |
| Glue-UX    | noindex |

📌 **의미**
“검색 시장 하나를 9~10개의 고품질 문서로 점유”

## 7. 실제 구현 예시 (미국 시장)

### 7.1 Inflation 클러스터

- **Entry:** “$1 in 1980 worth today”
- **Supporting:** 금액·연도 변형 질문 흡수
- **Depth:**
  - Purchasing Power
  - Wages
  - Housing
  - Education
  - $1 Million Case
  - CPI Explained
- **Glue-UX:** 계산기 (noindex)

👉 AI Overview에 인용될 수 있는 구조

### 7.2 Cost of Living 클러스터

- **Entry:** “How much do you need to earn to move states?”
- **Depth:**
  - CA vs TX
  - NY vs FL
  - Cheapest States
  - Most Expensive States
- **Glue-SEO:** 50 States Index
- **Glue-UX:** 비교 계산기

👉 계산기는 전환용
👉 검색 유입은 문서가 담당

## 8. 기술적 SEO & 최신 요소 반영

### 8.1 구조화 데이터 (선별 적용)

- Article
- FAQPage
- ItemList
- WebApplication (Glue-UX만)
- ❌ 폐지된 Rich Result 스키마 사용 안 함

### 8.2 Page Experience

- Core Web Vitals 기준 충족
- 모바일 퍼스트
- 빠른 LCP / 안정적 INP

👉 단, UX는 랭킹 보조 요소
👉 본질은 콘텐츠 품질

### 8.3 Indexability 전략

- noindex 페이지 sitemap 완전 제외
- canonical self-reference 고정
- crawl budget을 핵심 문서에 집중

## 9. 수익 모델 (Monetization)

### 9.1 1차 수익

- Google AdSense (미국 RPM 중심)

### 9.2 왜 광고 모델이 유리한가

- Finance / Tax / Retirement → 고 CPC
- Depth 콘텐츠 → 체류시간 길음
- 전환·CS·영업 비용 없음

**수익 공식**

> Revenue = Traffic × Pages/Session × RPM

Query-Driven 구조 → 세 항 모두 상승
